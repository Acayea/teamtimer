# src/db/migrate.ts

Runs Drizzle ORM migrations against the local Expo SQLite database on app startup.

## What it does

Calls Drizzle's `migrate()` with the generated migrations bundle from `src/db/migrations/migrations.js`. Drizzle tracks which migrations have already been applied (via a `__drizzle_migrations` table) and only runs new ones.

## Exports

### `runMigrations(): Promise<void>`

Applies any pending database migrations. Must be called before any database queries are made — typically at app startup in `app/_layout.tsx`.

```typescript
import { runMigrations } from '@/db/migrate';

await runMigrations();
```

## Parameters

None.

## Return value

Resolves when all migrations have been applied successfully. Rejects if a migration fails.

## Side effects

- Creates or updates the `__drizzle_migrations` tracking table in `teamtimer.db`.
- Executes the SQL from each unapplied migration file in order.

## Notes

- The migrations bundle (`migrations.js`) is generated by `drizzle-kit generate` and must be regenerated whenever the schema changes.
- `allowJs: true` in the TypeScript config (inherited from `expo/tsconfig.base`) enables importing the `.js` migration file directly.
